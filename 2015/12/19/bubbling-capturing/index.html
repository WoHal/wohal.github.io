Bubbling and Capturing

2015-12-19

http://javascript.info/tutorial/bubbling-and-capturing
http://www.jb51.net/article/42492.htm

1. 事件

事件是一种在页面上操作的处理，例如点击页面上的一个button，就会进行绑定在这个button上的对应操作。
然而，当点击了页面上的一个DOM元素时，该DOM的父元素也可能触发click事件，即DOM元素产生了冒泡。

2.事件机制

事件捕获——父元素先触发，子元素后触发
事件冒泡——子元素先触发，父元素后触发

在早期浏览器中，Netscape中使用了事件捕获，Microsoft使用了事件冒泡。

在W3C模型中将两者进行了合并，即任何事件发生时，都是先从DOM顶层开始进行事件捕获，直到事件源元素，然后再从事件源向上进行事件冒泡。

在IE9以下版本(不包含IE9)，只支持事件冒泡，
然后在其他浏览器都是先进行事件捕获，再进行事件冒泡。

需要注意的是，IE9以下版本不支持addEventListener绑定事件，而是用attachEvent绑定事件，在使用addEventListener进行事件绑定时，如果不设置useCapture，事件的触发默认发生在冒泡阶段；当设置useCapture为true时，事件的触发发生在捕获阶段。

3. 事件冒泡的顺序

IE<9：

div -> body -> html -> document

IE>=9 FF others：

div -> body -> html -> document -> window

4. 阻止事件冒泡

为了阻止事件冒泡，
W3C标准下：

event.stopPropagation()

IE<9：

event.cancelBubble = true
